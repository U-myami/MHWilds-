# モンスターハンターワイルズ 攻略サイト制作仕様書

## 1. プロジェクト概要
多様なプラットフォーム（コンシューマー、ソーシャル、アーケード等）のゲーム攻略に対応できるサイト基盤を構築する。過去の開発における失敗経験を糧とし、ポートフォリオとしての完成度を追求する。

## 2. ターゲット設計（ペルソナ）
ユーザーの習熟度に合わせて3つの層を定義し、個別のニーズを充足させる。

| ターゲット | 特徴・ニーズ |
| :--- | :--- |
| 初心者 | 過去作未経験。武器選び、ストーリー進行手順の把握。 |
| 復帰勢 | 過去作経験あり。システム変更点、現環境の最適装備の把握。 |
| 上級者 | モンスターの耐久値・気絶値の把握、部位ごとのダメージ計算。 |

## 3. コンセプト・UI/UX方針
「ゲームの世界観と調和した使いやすさ」を最優先事項とする。

* **世界観の再現:** ゲーム内のキャンプエリアやキャラクター、背景を活かしたデザイン。
* **可変UI:** ボタン素材に木製や鉄製のバリエーションを持たせ、ランダムで切り替える演出を検討。
* **情報の取捨選択:** ユーザーが必要な条件で絞り込むことで、余計な情報を排除する「見やすさ特化」の設計。
* **リッチコンテンツ:** 静止画よりも、実際のゲーム体験が伝わりやすい動画コンテンツを重視する。

## 4. サイト構造（サイトマップ）
サイトは以下の階層構造で設計する。

1. トップページ
2. よく見るページ（ショートカット）
3. 初心者ガイド
4. モンスター一覧
    * 出現場所、特性、弱点部位、属性相性、ドロップ確率、コメント欄
5. 装備一覧
    * 武器・防具の特性、強化、スキル、コメント欄
6. マップ情報
    * 環境ギミック、採取素材、コメント欄
7. マルチプレイ/フレンド募集
8. サポートキャラ紹介

## 5. データ構造定義 (JSON形式)
各データは拡張性の高いJSON形式で管理する。

### 5.1 モンスターデータ (`monster.json`)
ID、属性、出現場所、弱点倍率、ドロップ素材、ステータス（HP、気絶値等）を保持する。

### 5.2 装備・スキルデータ
* **武器 (`weapon.json`):** 武器種、レア度、属性値、攻撃力、会心率、斬れ味（数値管理）、スロット情報。
* **防具 (`armor.json`):** 部位、防御力（段階別）、属性耐性、付随スキル、シリーズID。
* **スキル (`skill.json`):** レベルごとの具体的な効果（攻撃力上昇値、会心率上昇値など）を定義。
* **シリーズ効果 (`series.json`):** 必要パーツ数に応じた発動効果の定義。
* **装飾品 (`ornament.json`):** スロットサイズと対応スキルの紐付け。

## 6. 技術選定・非機能要件
### 技術スタック
* **言語:** HTML, CSS, JavaScript
* **データ:** JSON, localStorage（ユーザーデータ保存用）

### パフォーマンス・品質管理
* **画像最適化:** WebP形式を採用し、1枚あたり300KB以下に抑制。
* **表示速度:** SEOの観点から表示速度の最適化を行い、モバイルフレンドリー（レスポンシブ）を徹底。
* **セキュリティ:** データの破損防止策およびセキュリティ対策の実施。
* **運用:** バージョン管理（Git等）を行い、アップデートごとの更新履歴を表示。
